# Muse App 诊断报告

## 问题现状

根据你的描述，当前存在以下问题：

1. **信号检测不显示** - 信号质量指示器显示"未连接/等待数据"
2. **Theta 脑波波段图没有数据** - 波形图显示"等待数据..."
3. **数据采集表格为空** - 没有接收到 EEG 数据包

## 根本原因分析

### 问题 1: BLE 数据通道未成功订阅

代码中已经添加了详细的调试日志，通过日志可以诊断：

```typescript
// 在 startMuseProtocol 函数中
addLog('🛡️ 订阅 EEG + PPG 通道…');
addLog(`📋 ATHENA_CHANNELS: ${ATHENA_CHANNELS.join(', ')}`);
```

**可能的原因：**
- BLE 服务发现未完成就尝试订阅特征值
- 特征值 UUID 不匹配
- 设备固件版本差异导致特征值不可用

### 问题 2: 数据解码失败

即使收到数据，也可能在解码阶段失败：

```typescript
const handleMuseDataPacket = (channel: string, base64Data: string) => {
  addLog(`📦 [${channel}] 收到数据: ${base64Data.substring(0, 30)}...`);
  
  if (channel === '0004' || channel === '0005') {
    try {
      const samples = decodeEEG(base64Data);
      addLog(`✅ [${channel}] 解码成功: ${samples.length}个样本`);
    } catch (e) {
      addLog(`❌ [${channel}] 解码失败: ${e}`);
    }
  }
}
```

## 诊断步骤

### 第一步：检查连接日志

连接 Muse 头环后，查看日志区域，应该看到：

```
✅ 发现 X 个 Characteristic
🛡️ 订阅 EEG + PPG 通道…
📋 ATHENA_CHANNELS: ...
🔍 尝试订阅通道: 0003 (UUID: ...)
✅ 成功订阅: 0003
...
📡 已订阅 7/7 个通道
🏁 数据流启动（高速模式）
```

**如果看到：**
- `⏭️ 跳过不可用通道` - 说明某些通道在设备上不存在
- `❌ 订阅失败` - 说明订阅过程出错
- `已订阅 0/7 个通道` - 说明所有通道都订阅失败

### 第二步：检查数据接收

成功订阅后，应该持续看到：

```
📊 [0004] 收到数据包，长度=XX
📦 [0004] 收到数据: AAAA... (XX字节)
✅ [0004] 解码成功: 12个样本
📈 processThetaWave: 处理 12 个样本
📊 波形缓冲区更新: 总长度=XX
```

**如果没有看到这些日志：**
- 数据通道订阅失败
- 设备未发送数据（可能需要重新发送 dc001 命令）

### 第三步：检查信号质量

应该看到：

```
📡 Control 通道收到数据: ...
🔋 电池: XX%
👤 佩戴质量 - TP9:XX% AF7:XX% AF8:XX% TP10:XX%
```

**如果没有看到：**
- Control 通道订阅失败
- 设备未发送状态信息

## 快速修复方案

### 方案 A：增加 BLE 初始化等待时间

代码中已经添加了 1.5 秒等待：

```typescript
addLog('⏳ 等待 BLE 栈完成服务枚举…');
await sleep(1500);
```

**如果仍然失败，可以尝试：**
1. 增加到 2500ms
2. 在订阅每个通道之间添加 100ms 延迟

### 方案 B：验证设备固件版本

不同的 Muse 设备（Muse 2, Muse S, Muse S Gen 2）可能有不同的特征值布局。

**检查方法：**
查看日志中的 `✅ 发现 X 个 Characteristic`，如果数量不是 7 个，说明设备特征值布局不同。

### 方案 C：手动触发数据流

有时需要多次发送 dc001 命令：

```typescript
// 尝试发送 3 次而不是 2 次
await write(device, CMD_START);
await sleep(150);
await write(device, CMD_START);
await sleep(150);
await write(device, CMD_START);
```

## 常见问题

### Q: 为什么电池显示正常但没有脑波数据？

A: Control 通道和数据通道是独立的。电池信息来自 Control 通道，脑波数据来自 ATHENA_CHANNELS。可能是数据通道订阅失败。

### Q: 日志显示"成功订阅"但仍然没有数据？

A: 可能是：
1. 设备未启动数据流（dc001 命令未生效）
2. 数据通道的回调函数未被触发
3. 数据解码失败

### Q: 如何确认设备正在发送数据？

A: 查看日志中是否有 `📊 [XXXX] 收到数据包` 的消息。如果有，说明设备在发送数据；如果没有，说明订阅或数据流启动有问题。

## 技术细节

### Muse BLE 协议

- **Service UUID**: `0000fe8d-0000-1000-8000-00805f9b34fb`
- **Control**: `273e0001-4c4d-454d-96be-f03bac821358`
- **EEG Channels**: 
  - 0003 (TP9)
  - 0004 (AF7) ← 用于 Theta 波形
  - 0005 (AF8) ← 用于 Theta 波形
  - 0006 (TP10)
  - 0007 (AUX)
- **PPG Channels**: 0010, 0011

### 数据流启动序列

1. `halt` - 停止当前操作
2. `p1035` - 设置高速模式
3. `dc001` × 2 - 启动数据流（必须发两次）
4. `s` - 心跳保活（每 30 秒）
